## 1. 研究の狙いが要求するシーン要件

「同一種類の剛体が段ボールにバラ積み → 上方1ショット計測 → 2.5D 片面点群の集団」を忠実に再現するには：

| ID | 要件 | なぜ必須か |
|----|------|-----------|
| RS1 | **容器（箱）に部品が収まる** | バラ積みの物理的拘束。周辺/隅の個体の見え方を決める |
| RS2 | **物理ドロップ＆山積み** — 同一剛体 N 個を重力落下→衝突→安定（積層・寄りかかり・部分埋没） | 姿勢分布（A2の接地バイアス）と**相互遮蔽**を生む本体 |
| RS3 | **単一トップダウン計測（ワンショット）** — 箱上方の1センサで深度1枚 → シーン全体の点群 | 「ワンショット」制約。各個体は自己遮蔽＋**相互遮蔽**＋斜め視で部分的になる |
| RS4 | **GT一式** — 個体姿勢 $T_i$ / 個体セグ（点→個体）/ 正準GTメッシュ / 観測和集合 $U$ | 評価ハーネス L1/L2/L3（[evaluation-harness.md](evaluation-harness.md)）の基準 |
| RS5 | **難易度の段階制御** — 物体形状クラス / 充填量 $N$ / 箱サイズ / ノイズ | アブレーション軸（被覆・対称・遮蔽・ノイズ） |

## 3. BlenderProc 検証結果（一次資料）

`bop_object_physics_positioning`（[doc](https://dlr-rm.github.io/BlenderProc/examples/datasets/bop_object_physics_positioning/README.html)）：
- **可**: 6面で囲った容器＋重力落下＋衝突settling（`simulate_physics_and_fix_final_poses`）。深度・姿勢・個体セグを BOP 形式で出力。物理山積み・相互遮蔽・GT は再現できる。
- **要設定（能力不足ではない）**: ① 容器が 4m³ 部屋 → 箱寸法に設定、② 既定は異種10物体 → **同一物体をN複製**、③ カメラはシェルサンプリング → **単一トップダウンに固定**。
- **コスト**: Blender 同梱で重く、フォトリアル RGB が主眼。点群/深度だけなら過剰になりがち。

## 4. 生成器アーキテクチャ（★要確認）

本研究は**深度/点群が主・RGB は当面不要**。この前提で2系統＋段階運用を比較：

| 構成 | 物理（piling） | 計測（深度・遮蔽・セグ） | 長所 | 短所 |
|------|---------------|--------------------------|------|------|
| **(A) BlenderProc 中心** | Blender Bullet | フォトリアルRGB＋深度＋BOPセグ | RGB・BOP実部品直結・sim-to-real | 重い・RGB過剰・反復遅い |
| **(B) PyBullet＋Open3D** | PyBullet | **Open3D RaycastingScene**（単一カメラ全シーンレイキャスト→深度・相互遮蔽・geometry-id セグ） | 軽量・全Python・幾何/深度特化・高速反復・全制御 | RGB無し・pybullet 要install・自作 |
| **(C) 段階運用 (B)→(A)** | — | — | 核を(B)で高速に、sim-to-real/実部品で(A)追加 | — |

**推奨: (C)。幾何コアは (B)、後段で必要に応じ (A)。** 理由: (B) は「箱内物理山積み（RS1/2）＋ワンショット単一視点深度（RS3, 相互遮蔽が自動的に正しく入る）＋GT一式（RS4）」を最小依存で実現でき、研究の核（参照無し groupwise registration）の反復に最適。Open3D RaycastingScene はローカルで利用可確認済み。(A) は RGB を使う sim-to-real や BOP 実部品メッシュ活用の段で価値が出る。

## 5. (B) PyBullet＋Open3D 生成器の具体設計

```
[物理] PyBullet
  1. 箱を構築（床＋4側壁の剛体、または tray メッシュ）。寸法は物体スケール L と充填量 N から決定。
  2. 同一メッシュを N 個ロード（非凸形状は VHACD 凸分解で衝突形状を作り貫通を防ぐ）。
  3. 箱上空のランダム位置・姿勢から順次/一括投下 → 重力＋衝突で settling。
  4. 静止後、各個体の GT 姿勢 T_i = getBasePositionAndOrientation を取得。

[計測] Open3D t.geometry.RaycastingScene（単一トップダウン）
  5. 全個体メッシュを world 姿勢 T_i で RaycastingScene に登録（geometry_id = 個体ID）。
  6. 箱上方に単一カメラ（透視 or 正射, 真下向き）。画素グリッドのレイを cast_rays。
  7. t_hit → 深度 → 世界点群。geometry_ids → 各点の個体セグGT（stage① スコープ外＝この GT を使う）。
     ※ 相互遮蔽・斜め視・自己遮蔽はレイキャストで自動的に正しく反映される。

[GT] T_i（physics）/ 個体セグ（ray geometry id）/ 正準GTメッシュ（既知）/ 観測和集合 U（被覆上界用）
[ノイズ] 深度に軸方向ガウシアン＋ドロップアウトを後付け（強度＝アブレーション軸, [synthetic-data-foundation.md](synthetic-data-foundation.md) §3.3）
```

- 既存の評価ハーネス・A0 融合（`src/partsrecon/eval`, `fusion`）は**そのまま接続可能**。生成器（`data/mock_bin.py`）を本 (B) 生成器に差し替える形。
- 出力フォーマットは現行 Scene と互換（partials は個体セグで分割、poses=T_i、s_gt=正準サンプル）に保つ。

## 6. 物体の難易度ラダー（「最初はうまくいく物体で確かめる」）

A1（参照無し registration）を**最初に通す**ための物体順序。早い段は「うまくいく」ことを確認し、後段で核心の困難をストレスする：

| 段 | 物体特性 | ストレスする課題 | 例 |
|----|---------|------------------|-----|
| 1（最初） | **非対称・特徴豊富・凸寄り** | なし（基準確認＝まず成功させる） | 段付きブロック / 非対称ブラケット |
| 2 | 弱対称・中凹凸 | 軽い姿勢曖昧性 | L字 / ヒンジ部品 |
| 3 | **強対称**（回転体・角柱） | 姿勢多義性（baseline F2） | ワッシャ / 円筒 / 直方体 |
| 4 | 特徴乏しい平滑面 | FPFH 失敗（F3） | 平板 / 球近傍 |
| 5 | 薄板・微小 | 法線・分解能（F4） | 薄板金 |

- **A0（oracle）は対称物体でも可**（姿勢を使わない）が、**A1 は段1（非対称・特徴豊富）から開始**する。
- ⚠️ A0 で使った **cuboid は段3（強対称）**＝A1 初手には不適。A1 最初の物体は**非対称形状**（段付きブロック等）を使う。
- 各段で**被覆 A2 が成立する形状**を選び、レジストレーション誤差を被覆限界から分離（[evaluation-harness.md](evaluation-harness.md) §6）。

## 7. 評価フローとの接続

1. (B) 生成器でバラ積みシーン（深度点群＋個体セグGT＋$T_i$＋正準GT）を作る。
2. 個体セグGTで各個体の部分点群を切り出す（stage① はスコープ外＝GT利用）。
3. A1 が $\hat T_i$ を推定 → 融合 → $\hat S$。
4. 評価ハーネスで L1（再構成）/ L2（姿勢, 対称群考慮）/ L3（被覆・誤差3層分解）。
5. 段1物体で「まず成功」を確認 → ラダーを上げて限界を定量化。
