> [research-overview.md](research-overview.md) §4 のファミリ A。比較探索の**アンカー（基準実装）**であり、それ自体が有力な解候補でもある。学習系（B〜E）はこのベースラインに対する改善として評価する。

## 1. 位置づけと方針

- **役割**: 学習を一切使わず、特徴量ベースのグローバル位置合わせ＋ポーズグラフ最適化＋融合＋表面再構成だけで全周囲点群 $\hat S$ を立ち上げる。解釈可能で、最短でエンドツーエンドが通る。
- **設計思想**: [research-overview.md](research-overview.md) の共通パイプライン（①〜⑥）のうち、stage① は**スコープ外（GTセグ利用）**。本メモは **②前処理 / ③個体間正準位置合わせ / ④融合 / ⑤表現 / ⑥出力** を定義する。
- **核心は ③**。後述の通り、本問題は通常のマルチビュー統合と2点で本質的に異なる（§4.0）。

### 入出力
- 入力: 個体ごとの部分点群 $\{P_i\}_{i=1}^N$（GTセグ済み、計量スケール既知、各ビューはセンサ視点が既知＝法線の向き付けに利用）。
- 出力: 正準座標系の全周囲統合点群 $\hat S$ ＋ 推定個体姿勢 $\{\hat T_i\}$。

---

## 2. stage② 前処理（各 $P_i$）

| 処理 | 内容 | 既定 | ライブラリ |
|------|------|------|-----------|
| ボクセルダウンサンプル | 計算量と密度の均一化 | $\delta = $ 物体スケール $L$ の $0.5\%$ | Open3D |
| 法線推定＋向き付け | **センサ視点方向へ向ける**（2.5Dなので一意に決まる） | k近傍 or 半径 | Open3D `estimate_normals` + `orient_normals_towards_camera_location` |
| 外れ値除去 | 距離計測の飛び点除去 | statistical outlier removal | Open3D |
| 特徴量 | グローバル位置合わせ用 | **FPFH**（半径 $\approx 5\delta$） | Open3D `compute_fpfh_feature` |

> 法線の一貫した向き付けは、後段の point-to-plane ICP と Poisson 再構成の品質を左右する重要前処理。

---

## 3. stage③ 個体間正準位置合わせ ★中核

### 3.0 本問題特有の難しさ（設計の前提）
1. **重複グラフの連結性問題**: 視点（個体の安定姿勢）は制御できない。2つの部分ビューが物体の**反対側**を見ていれば重複ゼロで直接位置合わせ不能。全ビューを共通系へ繋ぐには、**重複するビュー対の連鎖**でグラフが連結している必要がある。これは被覆性 A2（全周囲が観測されるか）とは**別物の前提**（A2 が成立しても重複グラフが分断されうる）。
2. **対称性による多義性**: FA部品は対称形状が多く、ペア位置合わせ $T_{ij}$ が対称群の分だけ多義化 → ループ不整合を生む。

→ 設計はこの2点への耐性を中心に据える。

### 3.1 アルゴリズム（推奨パイプライン）
参照モデル無しの groupwise registration を、**ペア粗位置合わせ → 重複しきい値でポーズグラフ構築 → グローバル最適化 → インクリメンタル成長 → 群ICP精緻化** で組む（古典「複数レンジ画像の自動統合」系の系譜）。

```
1. ペア粗位置合わせ（重複のある対のみ生かす）
   for 候補ペア (i,j):
     T_ij ← RANSAC global registration（FPFH特徴マッチング, 部分重複対応）
            ├ ロバスト代替: TEASER++（高外れ値耐性, 低重複に強い）
            └ 特徴が効かない平滑/対称形状: 4PCS/Super4PCS（特徴フリー）
     T_ij ← point-to-plane ICP で精緻化
     記録: fitness, inlier_rmse, inlier_ratio
2. ポーズグラフ構築
   ノード = 各ビュー
   エッジ = fitness/inlier_ratio がしきい値超のペア（= 重複が確かな対のみ）
   不確実エッジは line-process でロバスト化（誤エッジを最適化中に棄却）
3. グローバルポーズグラフ最適化（Levenberg-Marquardt, ループ整合）
   → グローバル姿勢 {T̂_i}
4. インクリメンタル成長（SfM的・任意）
   良重複のシードペアからモデルを成長させ、次ビューを成長モデルへ登録
   → 重複が間接的にしか繋がらない場合の連結性を補強
5. 群ICP（groupwise ICP）精緻化
   全ビューを暫定モデルへ反復整合（モデル↔姿勢を交互更新, EM的）
```

- **候補ペアの枝刈り**: $N$ が小なら全 $\binom{N}{2}$ ペア（$N{=}20$ で190対）で十分。大 $N$ はグローバル記述子の類似度や乱択でペアを絞る（$O(N^2)$ 回避、§7 未決）。
- **連結性チェック**: ポーズグラフが**単一連結成分**になるか検査。分断時は「未観測/低重複により全周囲が繋がらない」と明示警告し、評価で被覆限界（[evaluation-harness.md](evaluation-harness.md) §6.1）と区別して報告。

### 3.2 対称性の扱い
- ペアエッジを対称群分の**多重仮説**として保持し、グローバル最適化＋ループ整合＋ line-process に正解を選ばせる。
- 強対称（回転体等）はペア位置合わせが原理的に縮退 → **本ベースラインの主要弱点**。ここが学習系（B: 正準化, C: 陰関数同時最適化）が優位を取りうる箇所として記録（比較の見どころ）。
- 対称群は合成では GT 既知（BOP `models_info.json`）を利用可。

---

## 4. stage④ 融合

```
1. 全ビューを正準系へ: ∪_i T̂_i^{-1}(P_i) を連結
2. 冗長統合: ボクセルダウンサンプル（サイズ δ）で重複点をマージ
3. クリーニング: statistical/radius outlier removal で誤登録の浮き点を除去
4. 法線統合: 近傍で平均化・再向き付け
```

---

## 5. stage⑤ 表現・補完（古典）

- **既定（主出力）**: 融合・クリーニング後の点群をそのまま $\hat S$ とする（補完なし）。未観測領域を**捏造しない**＝ Accuracy 指標で不利にならない。
- **オプション variant**: Screened Poisson 表面再構成 → メッシュ → 標準密度で再サンプル。
  - ただし Poisson は穴を外挿で埋め**未観測領域に表面を捏造**する。→ **density-based trimming**（低密度頂点を除去）で観測領域外の捏造を抑制してから採用。
  - 「点群直出力」と「Poisson variant」の両方を評価し、補完の功罪を可視化。

---

## 6. stage⑥ 出力と正準系の固定

- $\hat S$（点群）＋ $\{\hat T_i\}$ を出力。
- 内部正準系は再現性のため決定的に固定（融合点群の PCA 主軸＋重心原点）。対称形状で PCA 軸は曖昧だが、**評価ハーネスがグローバル整列で吸収**（[evaluation-harness.md](evaluation-harness.md) §3）するため内部規約で可。

## 7. 既知の限界・失敗モード（要記録）

| # | 失敗モード | 原因 | 緩和・切り分け |
|---|-----------|------|----------------|
| F1 | 重複グラフ分断 | 安定姿勢の多様性不足で繋ぐ重複が無い | 連結性チェック＋警告。被覆限界と分離報告。インクリメンタル成長で補強 |
| F2 | 対称物体で位置合わせ縮退 | 対称群による多義性 | 多重仮説＋ロバスト最適化。根本解は学習系（比較の論点） |
| F3 | 平滑・特徴乏しい面で FPFH 失敗 | 幾何特徴が出ない | TEASER++/4PCS へフォールバック |
| F4 | 薄板・微小部品で法線/位置合わせ不安定 | 計測分解能・自己遮蔽 | δ・半径の調整、ノイズアブレーションで感度評価 |
| F5 | Poisson が未観測領域を捏造 | 外挿補完 | density trimming、点群直出力を主に |
| F6 | $O(N^2)$ ペアで大 $N$ 不可 | 全ペア位置合わせ | ペア枝刈り（記述子類似・乱択） |

→ これらは [research-overview.md](research-overview.md) §8 のリスク表とも対応。F1（重複連結性）は**本問題特有**で、論文での重要な議論点。

## 8. 評価ハーネスとの接続・段階的実装

[evaluation-harness.md](evaluation-harness.md) §6 の誤差3層分解に**実装段階を一致**させると、各段で何が効くか切り分けられる：

| 段階 | 内容 | 検証する層 | 備考 |
|------|------|-----------|------|
| **A0（oracle融合）** | GT姿勢 $\{T_i\}$ で融合＋再構成（③をスキップ） | ①被覆限界 + ③表現誤差の上界 | [synthetic-data-foundation.md](synthetic-data-foundation.md) §7 の PoC と同一。基盤検証も兼ねる |
| **A1（ペア＋ポーズグラフ）** | §3.1 の 1〜3 | ②レジストレーション誤差 | 最小の本物 groupwise registration |
| **A2（成長＋群ICP）** | §3.1 の 4〜5 を追加 | ②の改善 | 連結性・精度の上積み |

- 各段で oracle（A0 = 上界）を併記し、A1/A2 の「伸びしろ」を提示。
- アブレーション軸（$N$ / 対称性 / 遮蔽 / ノイズ）で A1/A2 の頑健性を測る。

## 9. ハイパーパラメータ既定（初期値）

| 記号 | 意味 | 既定 |
|------|------|------|
| $\delta$ | ボクセル/位置合わせ解像度 | $0.5\%\,L$ |
| FPFH 半径 | 特徴計算近傍 | $\approx 5\delta$ |
| RANSAC | 反復・距離しきい | Open3D 既定ベース、距離 $\approx 1.5\delta$ |
| エッジ採用しきい | fitness / inlier_ratio | 初期 $0.3$ / $0.25$（要チューニング） |
| ICP | 種類・距離 | point-to-plane、距離 $\approx 2\delta$ |
| Poisson | depth / trim | depth 8〜9、density 下位 trim |

---

## 10. 実装計画（最短経路）

1. **A0 を最初に** ✅ 完了（モックデータ、[poc-a0-results.md](poc-a0-results.md)）: GT姿勢で融合→評価ハーネスを通し、基盤(R1–R5)・指標・被覆性上界を確認。実データ化(BlenderProc)は M2。
2. **A1**: 前処理→ペアRANSAC+ICP→ポーズグラフ→グローバル最適化。Open3D の multiway registration レシピを土台に、**重複しきい・連結性チェック・対称多重仮説**を本問題向けに追加。
3. **A2**: インクリメンタル成長＋群ICP を上積み。
4. アブレーションと oracle 比較で限界（§7）を定量化 → 学習系（B/C…）選定の根拠にする。

## 11. 未決事項

| 未決 | 内容 | 確定タイミング |
|------|------|----------------|
| ロバスト位置合わせの主軸 | RANSAC か TEASER++ か（低重複・対称耐性） | 文献調査＋A1実験で比較 |
| 特徴フリー併用 | 4PCS/Super4PCS をいつ使うか | 平滑/対称物体で FPFH 失敗を確認したら |

| 対称検出 | GT利用か自動推定か | 実機フェーズで GT 無し時に必要 |
| Poisson trim 閾 | 捏造抑制の最適 trim 量 | A0 実験で調整 |
