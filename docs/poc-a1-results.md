# A1 PoC 結果 — 参照無し groupwise registration の feasibility（Stanford Bunny）

> 本研究の核（「姿勢なし SfM」相当 = 参照モデル無しで部分ビュー群を共通正準系へ）の最初の実証。複雑・非対称な Bunny 1種類で「本当に通るか」を A0(GT姿勢=上界)と比較。
>
> - ステータス: 素朴A1は遮蔽下で失敗 → **(a) 3-cycle整合エッジゲートで 11/12 登録・CD 2.05%・F@1%=0.87 = 実現可能性 確認**（§5, §6）
> - 実施日: 2026-06-24
> - コード: `src/partsrecon/registration/groupwise.py`, `scripts/run_a1_poc.py`
> - 再現: `python scripts/run_a1_poc.py --n 12 --seed 0`

---

## 1. 結果（Bunny, N=12, L=192.7mm）

| 指標 | A0 oracle（GT姿勢=上界） | A1（参照無し登録） |
|------|---:|---:|
| CD (%L) | **0.45** | **10.88** |
| Accuracy (%L) | 0.23 | 6.73 |
| Completeness (%L) | 0.22 | 4.15 |
| F@1%L | 0.989 | 0.161 |
| F@2%L | 0.995 | 0.295 |

- 登録: 12/12 を主連結成分に入れた（採用エッジ66 / tree 11）が、**姿勢一貫性は回転誤差 中央値 124°・最大 173°**、正しく登録できたのは **5/12（<10°）**。

## 2. 診断

1. **情報は十分** — A0 oracle が CD 0.45%・F@1%=0.99。部分ビューの和集合は全周囲を復元できる（データは足りている）。
2. **ボトルネックは参照無し登録（誤差3層分解の②層）** — 各個体は**上面側の部分点群のみ**で、姿勢が異なる個体間は**重複が乏しい**。FPFH+RANSAC が**低重複でも“それらしい”誤対応を捏造**し、誤エッジがポーズグラフを汚染 → 大域最適化が誤収束。
3. これは [baseline-classical-geometry.md](baseline-classical-geometry.md) §7 の **F1(重複連結性)＋F3(特徴失敗)** が顕在化したもの。M1.5 の FurnSet/SGHR 等が示す難所と一致。
4. **パイプライン自体は正しい** — 座標規約は代数的に検証済み、かつ 5/12 は正しく登録できている。問題は**誤エッジの混入**であって実装バグではない。

## 3. 含意

- 「同一インスタンス群を未知物体のマルチビュー代用にする」着想は **A0 で原理的に成立**するが、**素朴な古典登録では実現できない**。参照無し・低重複・部分ビューという条件が本質的に難しい（ユーザー予測どおり）。
- **この負の結果が研究の主戦場を特定した**：頑健な参照無し multiway registration（誤エッジ排除）が鍵。

## 4. 次の選択肢

| 方向 | 内容 | 根拠 |
|------|------|------|
| **(a) 古典の頑健化** | 重複推定でエッジを事前ゲート / RANSAC 検証強化・相互フィルタ / TEASER++ / SGHR 流の「信頼グラフ初期化＋reweighting」 | [literature-survey.md](literature-survey.md) §3, §10。誤エッジ排除が核 |
| **(b) 学習系へ** | ファミリ B(正準化: ConDor) / C(陰関数同時最適化: DeepSDF系) | 低重複でも姿勢を出せる可能性。literature §5 |
| **(c) カリキュラム** | まず重複が多い条件（少数・分離配置 or ほぼ全周ビュー）で A1 成功を確認 → 段階的に難化 | パイプライン健全性の確認と、どの重複量で破綻するかの境界測定 |

> 推奨初手: **(c) で「どの重複量・N で破綻するか」の境界を測りつつ、(a) のエッジゲートで誤エッジを落とす**。それで届かなければ (b) 学習系へ。

## 5. (c) カリキュラム — 重複量の掃引（クリーン合成ビュー）

`src/partsrecon/data/synthetic_views.py`（法線キャップで可視量を制御）+ `scripts/run_a1_curriculum.py`。Bunny N=10、`cap_angle` を 180°(全周)→80°(半球以下)に掃引、**GT既知ゲージで評価**:

| cap° | 登録正解 | rotMed | CD%L | F@1 | 被覆% |
|------|---------|--------|------|-----|------|
| 180–110 | 10/10 | ~0° | 0.5–0.65 | 0.998+ | ~100 |
| 95 | 9/10 | 0.5° | 1.53 | 0.882 | 100 |
| 80 | 9/10 | 0.6° | 2.05 | 0.857 | 97.5 |

**発見:**
1. **クリーンな部分ビューなら A1 は cap=80°まで成功** → パイプライン健全性を確認、かつ**「見えの小ささ」単体は破綻要因でない**。
2. バラ積みでの失敗の真因は**遮蔽（自己＋個体間）由来の誤対応**と判明。
3. **評価の教訓（2バグ修正）**: ① RANSAC整列(`global_align`)は**部分モデルで誤収束**し CD を過大化 → **GT既知ゲージ（root の `inv(T_gt[root])`）で整列**して測る。② 姿勢一貫性の基準は **root に統一**（`ids[0]` だと root と不一致で誤判定）。

## 6. (a) 3-cycle 整合エッジゲート → バラ積みで実現可能性を確認

`groupwise.py` に **3-cycle 整合フィルタ**（i→j→k と i→k が一致しないエッジを除去、precision優先）を追加。バラ積み Bunny N=12:

| 指標 | 素朴A1 | **+サイクルゲート(a)** | A0 oracle |
|------|-------:|----------------------:|----------:|
| 採用エッジ | 66 | **43** | — |
| 登録正解(<10°) | 5/12 | **11/12** | — |
| 回転誤差 中央値 | 124° | **0.85°** | — |
| CD (%L) | 10.9 | **2.05** | 0.45 |
| F@1%L | 0.16 | **0.874** | 0.989 |

**結論:** 素朴な参照無し登録は遮蔽下で破綻するが、**誤エッジを 3-cycle 整合で落とすと 11/12 を正しく登録し、ほぼ全周囲モデル（F@1%=0.87・完全性 0.43% L）を復元**。本研究の核（姿勢なし SfM 的な参照無し復元）は **実現可能**と確認。

**残課題:**
- (i) 最後の 1/12 の誤登録（Accuracy を 1.6% へ悪化）→ 構築モデルへの個体再検証で除去。
- (ii) RANSAC 非決定性で run 間にばらつき（同条件で 11/12〜5/12 まで変動）→ seed 固定 / 多重開始 / より強い検証。
- (iii) 対称・特徴乏物体、N 拡大、センサノイズでの頑健性。

## 7. 前提保証シーンでの A1 — ほぼ完璧に全周囲復元

**前提（ユーザー指定）**: 「マッチングに有効な大きさの部分セグメントがシーン内に網羅され、すべて適切に組み合わせれば全周囲点群になる」。これを**生成器で保証**する `src/partsrecon/data/synthetic_views.py::make_covering_scene`:
1. 各セグメントはマッチング有効な大きさ（`cap_angle=120°`＝半球超、`min_seg_points` 未満の小断片は除外）。
2. セグメント和集合が全周囲を被覆（s_gt→和集合の最近傍で被覆率を逐次検証し、`coverage_target`=99% 達成で停止）。
3. cap>半球なので任意 2 ビューが重複 → 登録グラフ連結（「組み合わせ可能」を保証）。

Bunny、自動 N=10（被覆 **100%** で停止）:

| 指標 | A0 oracle | A1（参照無し +(a)） |
|------|----------:|--------------------:|
| 登録正解 | — | **10/10**（回転誤差中央値 **0.10°**） |
| CD (%L) | 0.527 | **0.549** |
| F@1%L | 0.997 | **0.999** |

**結論:** 前提が満たされれば、A1 は全セグメントを正しく組み合わせ **oracle 同等の全周囲点群を復元**。本研究の中核仮説が、前提を明示的に保証した条件で**成立**することを実証。

実行: `run_a1_poc.py --scene covering`（既定＝前提保証）／ `--scene bin`（物理バラ積み＝遮蔽で前提が崩れうる難条件、11/12）。

## 8. (A) 物体クラス × ブリットルネス検証（FGR × 5シード）

`scripts/run_a1_brittleness.py`（公開DBメッシュ＋FAプリミティブ10種、前提保証シーン、評価＝姿勢自由・多重ICP整列の重なり率 F@1、5シードの mean±std）。律速回避: `n_max=12`＋`max_pairs`上限、逐次出力＋resume 対応。

| object | 種別 | F@1 mean±std | 分類 |
|--------|------|:---:|------|
| ellipsoid | 平滑曲面 | 0.99±0.01 | **安定OK** |
| cylinder | 回転対称 | 0.72±0.15 | 脆い |
| bunny | 曲面・特徴豊富 | 0.63±0.32 | 脆い |
| washer | 回転対称・穴 | 0.63±0.23 | 脆い |
| cone | 回転対称+頂点 | 0.63±0.24 | 脆い |
| knot (公開) | 滑らか曲面 | 0.60±0.31 | 脆い |
| cuboid | 離散対称・平面 | 0.55±0.31 | 脆い |
| armadillo (公開) | 曲面・特徴豊富 | 0.42±0.23 | 脆い |
| plate | 薄板・平面 | 0.42±0.09 | **安定NG** |
| lblock | 平面・角 | 0.24±0.07 | **安定NG** |

**結論（baselineの限界 → 学習移行の動機）:**
1. **FGR（高速）は脆い** — 安定OKは ellipsoid のみ。bunny/armadillo/knot など曲面・特徴豊富な物体ですら F@1 が 0.2〜1.0 で乱高下（std~0.3）。シード（バラ積みの引き）次第で成否が変わり、**信頼できない**。
2. **平面系（lblock/plate）は安定してNG** — 平面どうしを特徴で識別できず誤って貼り合わせる（F~0.2〜0.4）。FA頻出の板金が最弱。
3. **RANSAC（堅牢）は安定方向だが遅すぎ** — 物体あたり分単位（§6の bin で 11/12 を出したが計算は重い）。
4. → 古典は **「速い＝脆い（FGR）／堅牢＝遅い（RANSAC）」のジレンマ**。前提（マッチング有効な網羅セグメント）を保証しても解けない物体クラスが残る → **学習ベース（ファミリB/C; 学習特徴で平面・対称をロバストに、feed-forwardで高速）への移行を動機づける**。

> 注: 上表は FGR・5シード・並列(ProcessPool)版の暫定値。並列は Open3D 内部マルチコアと競合して**速度に逆効果**だったため廃止し、§9 に**逐次・同条件**の確定比較を置く。

## 9. RANSAC vs FGR 同条件比較（3シード・逐次・rv=0.015）★確定

`scripts/run_a1_brittleness.py --method {ransac,fgr} --seeds 0 1 2`（**逐次実行**＝各タスクで Open3D が全コア使用。ProcessPool 並列は過剰サブスクで逆効果のため廃止）。評価＝姿勢自由・多重ICP整列の重なり率 F@1（mean±std）、per-task 実測秒。

| 物体 | 種別 | RANSAC F±std (秒) | FGR F±std (秒) |
|------|------|:---:|:---:|
| bunny | 曲面・特徴豊富 | **0.998±0.000** (36) | 0.642±0.252 (32) |
| armadillo (公開) | 曲面・特徴豊富 | **0.998±0.000** (39) | 0.303±0.091 (30) |
| ellipsoid | 平滑曲面 | 0.997±0.001 (58) | 0.998±0.002 (32) |
| knot (公開) | 滑らか曲面 | 0.750±0.328 (65) | 0.507±0.330 (47) |
| cone | 回転対称+頂点 | 0.782±0.116 (89) | 0.529±0.252 (46) |
| cylinder | 回転対称 | 0.483±0.229 (115) | 0.834±0.095 (62) |
| cuboid | 離散対称・平面 | 0.337±0.074 (47) | 0.790±0.169 (38) |
| washer | 回転対称・穴 | 0.653±0.313 (58) | 0.628±0.261 (31) |
| lblock | 平面・角 | 0.516±0.307 (57) | 0.185±0.020 (51) |
| plate | 薄板・平面 | 0.399±0.041 (31) | 0.324±0.023 (27) |

**平均 per-task: RANSAC ~59.5s / FGR ~39.6s**（FGR は約1.5倍速いが桁違いではない＝make_covering_scene と align_to_gt の方法非依存オーバーヘッドが効くため）。

**確定結論（baselineの限界 → 学習移行の動機）:**
1. **特徴豊富な曲面（bunny/armadillo/ellipsoid）**: RANSAC が安定して高F（0.998, std≈0）。FGR は bunny/armadillo で脆い。→ 堅牢だが遅い。
2. **FA頻出の対称・平面（cylinder/cone/cuboid/washer/knot/lblock/plate）**: **両手法とも高分散（脆い）or 失敗**、しかも**どちらが解けるかが手法でバラバラ**（cylinder/cuboid は FGR 優位、cone/bunny は RANSAC 優位）。**安定して解ける古典手法が存在しない**。plate は両者失敗。
3. **速度**: 両者とも ~40〜60s/task と遅い。

→ 古典は **「特徴豊富曲面のみ RANSAC で堅牢だが遅い／FA頻出の対称・平面は両手法とも信頼できない」**。学習移行の動機が、特に**対称・平面部品**で決定的。
（生データ: `outputs/a1_brittleness_ransac/` , `outputs/a1_brittleness_fgr/`）
